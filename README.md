# Multi-Echelon Inventory Optimization System

An end-to-end supply chain analytics and optimization project built to solve inventory planning across multiple echelons using demand forecasting, inventory policy design, optimization, and simulation.

---

## Project Overview

This system helps answer:

- How much inventory should be stocked?
- Where should inventory be placed across the supply chain?
- Which SKUs are critical?
- What is the stockout risk?
- How can holding cost be minimized while maintaining service levels?

Supply chain network modeled:

```text
Supplier → Factory → Central Warehouse → Regional Warehouse → Store
```

---

## Business Objective

Build an autonomous inventory control tower that can:

1. Forecast demand
2. Classify SKUs by importance
3. Calculate inventory policies
4. Optimize stock placement across echelons
5. Simulate stockout risk
6. Expose results through dashboards and APIs

### Phase 5: Demand Forecasting

Models built:

- Moving Average
- ARIMA
- XGBoost

- RMSE
- MAPE
- Bias

Run:

```bash

```

